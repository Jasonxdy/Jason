Hello, JSP world!

JSP (Java Server Page)란, Java 코드가 들어가 있는 HTML 코드를 말하며 서블릿의 복잡함을 간단하게 작성, 해결할 수 있다.
JSP의 가장 큰 장점은 Servlet에서 비즈니스 로직(Java)과 프레젠테이션 로직(HTML)을 분리할 수 있다는 것이다.
프로그래머가 직접 Servlet에 코딩한 것보다 더욱 최적화된 형태의 Servlet을 생성해줌.
웹 애플리케이션 상에서 변수의 사용 가능한 범위 설정이 쉽다.

JSP - 1

스크립팅 원소(Scripting Element)는 JSP 페이지에서 자바 코드를 직접 기술할 수 있게 해주는 기능.
1. <%! %> (선언문) : 멤버변수, 메소드를 선언하기 위해 사용된다.
2. <% %> (스크립틀릿, scriptlet) : JSP에서 자바코드 사용을 가능하게 해줌. 3. <%= %> (표현식, expression) : Java에서 작성한 변수, 결과값을 출력하기 위해 사용된다.

Servlet - 4 (Servlet + JSP)

Servlet은 응답 화면에 표현될 HTML 코드를 프로그램적으로 작성한다.
이로 인해 생기는 단점은 비즈니스 로직과 화면 구현 디자인 코드가 같이 있어 가독성이 떨어지고 개발이 어려워진다.
이러한 구조는 협업을 중요시 하는 개발자들의 입장에서 좋지 않은 코딩 방법으로 비즈니스 로직과 디자인은 분리되는 것이 좋다.
또한 Servlet에서 HTML 코드를 수정하는 경우 재 컴파일을 위해 서버를 재시동 해야하는 문제가 있고, HTML에는 서버에서 얻어온 정보를 동적으로 반영하기 힘들다.
